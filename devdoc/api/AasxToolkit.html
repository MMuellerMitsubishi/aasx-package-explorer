    Cli

    Handles the command-line interface.

    The handling of the command-line arguments was separated in three phases:

    • Command declaration: the interface is specified in a declarative manner
      • ,
    • Instruction parsing: the actual command-line arguments are parsed into instructions and
    • Execution: the chain of instructions is finally executed, one instruction at the time.

    Think of the process as a mapping: commands -> instructions -> execution.

    Declarative specification allows for better readability. Even without the help message, we aim to make it obvious how the program should be invoked by just looking at the code.

    The decoupling of parsing to a separate phase let us encapsulate all the low-level details into this module and test them in separation.

    By introducing instructions as units of execution, the code can be easily reused. For example, different commands can be parsed into the same instruction with different parameters.

    The execution phase allows us to nicely share the context between the execution of the individual instructions and return pre-emptively from the execution loop, if necessary.

    To add a new command-instruction-execution to the program, follow these steps:

    1. Start by conceptualizing what data your instruction needs. Write it down by writing a class implementing Cli.IInstruction and put it in AasxToolkit.Instruction.
    2. Add the command to Program. Provide parse function that translates command-line arguments into your instruction.
    3. Implement the execution of your instruction by adding a case corresponding to your instruction to the switch in Execution.
    4. Write the corresponding unit tests. Depending on the complexity, you might need to test the parsing and execution in isolation.
