    Namespace AasxDictionaryImport.Eclass

    Classes

    Class

    An eCl@ss class. eCl@ss has two types of classes: application classes and classification classes. Classification classes are part of the classification hierarchy while an application class defines the properties for a classification class. Unfortunately, the classification classes do not store references to their application classes. Therefore we manually compute this mapping in Context when loading the data from the XML file.

    In eCl@ss Basic, there is a 1:1 relation between classification and application classes. In eCl@ss Advanced, there may be several application classes per classification class (typically one for the basic attributes and one for the advanced attributes). As this implementation only supports eCl@ss Basic, we assume that there is at most one application class per classification class and just transfer the application class's properties to the classification class.

    Context

    Data context for eCl@ss data. The context loads the classes and properties from an XML document. It removes all deprecated elements because they are no longer part of the eCl@ss release. It also computes a mapping from classification classes to application classes. For more information on the class types, see Class.

    The main class structure is loaded from the English-language XML export. XML files for other languages and an XML file with information about the units can be loaded additionally.

    DataProvider

    Data provider for eCl@ss Basic data. The data is read from XML files using the OntoML scheme, see Context for more information. The complete eCl@ss exports use one file per language and domain. In this implementation, a data source is always backed by an English export file. If present, export files in other languages and unit files are loaded additionally.

    This data provider assumes that the eCl@ss XML files follow this naming convention: 

    • <Prefix>_<Version>_<Lang>_*.xml
      for class exports, and
    • <Prefix>_UnitsML_*.xml
      for unit data,
    where 
    <Prefix>
    is a constant prefix per eCl@ss release, 
    <Version>
    is either 
    BASIC
    or 
    ADVANCED
    and 
    <Lang>
    is an uppercase two-character language code (e. g. 
    EN
    for the English export).

    DataSource

    Data source for eCl@ss data. The eCl@ss data is read from XML files, and a data source represents one English eCl@ss XML file, optionally with additional XML files in other languages. For more information on the XML parser, see the Context and Element classes.

    Element

    An element in the eCl@ss data set, backed by an XML element from the exported XML document. Currently, we support Class and Property elements.

    Property

    An eCl@ss property. eCl@ss properties are assigned to application classes, see Class.
